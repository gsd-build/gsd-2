# Roadmap: GSD Mission Control

## Overview

GSD Mission Control goes from zero to a fully functional local dashboard in 10 phases. We start with the monorepo scaffold and Bun server, build the real-time file-to-UI pipeline, lay down the panel shell with the design system, then fill each panel with live data -- sidebar and milestone view first, then slice detail and active task. Chat with Claude Code integration follows as the core interaction model. Session flow and animations give the product its identity. Discuss and review modes add the structured interaction layers. Live preview brings the dev server into the window. Keyboard shortcuts and accessibility close it out.

### Phase 1: Monorepo + Bun Server Bootstrap

**Goal**: Developer can run `bun run dev` and see a served application at localhost:4000 from a properly structured monorepo workspace
**Depends on**: Nothing (first phase)
**Requirements**: MONO-01, MONO-02, MONO-03, SERV-01

### Phase 2: File-to-State Pipeline

**Goal**: Changes to `.planning/` files are detected, parsed into structured state, diffed, and pushed to connected browser clients in under 100ms
**Depends on**: Phase 1
**Requirements**: SERV-02, SERV-03, SERV-04, SERV-05, SERV-08, SERV-09
**Success Criteria** (what must be TRUE):

1. File watcher detects all file changes in `.planning/` recursively without race conditions on partial writes
2. State deriver parses `.planning/` files (ROADMAP.md, STATE.md, plans, config.json) into typed JSON state objects
3. WebSocket server on :4001 pushes diff-only updates to all connected clients within 100ms of file change
4. Client reconnects automatically with exponential backoff after disconnect, receiving full state on reconnect
5. Bun process restart reconstructs complete state from `.planning/` files with no data loss
   **Plans**: 3 plans

Plans:

- [ ] 02-01-PLAN.md -- Shared types, debounced file watcher, and state derivation engine
- [ ] 02-02-PLAN.md -- WebSocket server, state differ, and pipeline orchestrator
- [ ] 02-03-PLAN.md -- Client-side reconnecting WebSocket and usePlanningState React hook
