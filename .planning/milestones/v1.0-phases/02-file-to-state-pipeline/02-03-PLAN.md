1. Create `packages/mission-control/src/hooks/useReconnectingWebSocket.ts`:
   - Export a React hook `useReconnectingWebSocket(url: string)` that returns `{ status: "connecting" | "connected" | "disconnected", send: (msg: string) => void }`
   - Internal logic (NOT a class, just closure-based state in the hook):
     a. `connect()` function creates `new WebSocket(url)`
     b. `onopen`: set status to "connected", reset attempt counter to 0
     c. `onclose`: set status to "disconnected", schedule reconnect with exponential backoff
     d. `onerror`: let onclose handle reconnect (errors always trigger close)
     e. `onmessage`: forward to an onMessage callback (passed via options or ref)
   - Backoff formula: `delay = Math.min(1000 * Math.pow(2, attempt), 30000)`, jitter = `delay * 0.1 * Math.random()`
   - Use `useEffect` for lifecycle: connect on mount, close on unmount
   - Use `useRef` to hold WebSocket instance (avoids stale closure issues)
   - Use `useCallback` for send function

2. Create `packages/mission-control/src/hooks/usePlanningState.ts`:
   - Export `usePlanningState()` hook that returns `{ state: PlanningState | null, status: "connecting" | "connected" | "disconnected" }`
   - Internally uses `useReconnectingWebSocket("ws://localhost:4001")`
   - Maintains `lastProcessedSequence` ref
   - On message:
     a. Parse JSON
     b. If `msg.sequence <= lastProcessedSequence`, ignore (stale message)
     c. If `msg.type === "full"`: replace entire state, set lastProcessedSequence = msg.sequence
     d. If `msg.type === "diff"`: shallow merge `msg.changes` into current state, set lastProcessedSequence = msg.sequence
   - Uses `useState<PlanningState | null>(null)` for state storage
   - Import PlanningState and StateDiff types from `../server/types`

3. For tests, extract the backoff calculation and sequence filtering into pure functions (exported separately or as a testable module) so they can be tested without a browser WebSocket:

   - `shouldProcessMessage(messageSequence: number, lastProcessed: number): boolean`
   - `applyStateUpdate(currentState: PlanningState | null, message: StateDiff): PlanningState | null`

   Test these pure functions directly. Do NOT try to test React hooks or real WebSocket connections -- those are integration-tested in Plan 02's pipeline tests and verified in the human-verify checkpoint at phase end.
