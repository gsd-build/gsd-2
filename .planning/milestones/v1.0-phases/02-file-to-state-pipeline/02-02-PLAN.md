    - Test: WebSocket server accepts connections on :4001
    - Test: WebSocket server sends full state message on client connect
    - Test: WebSocket server broadcasts diff messages to all subscribed clients

  <action>
1. Create `packages/mission-control/src/server/differ.ts`:
   - Export `computeDiff(oldState: PlanningState, newState: PlanningState): StateDiff | null`
   - Compare each top-level key using `JSON.stringify(oldState[key]) !== JSON.stringify(newState[key])`
   - Return null if no changes detected
   - Return `{ type: "diff", changes: { ...changedKeys }, timestamp: Date.now(), sequence: 0 }` (caller sets sequence)
   - This is intentionally simple -- for 5-6 top-level keys, JSON.stringify comparison takes <1ms

2. Create `packages/mission-control/src/server/ws-server.ts`:
   - Export `createWsServer(options: { port: number, getFullState: () => PlanningState }): WsServer`
   - Use `Bun.serve()` with WebSocket support:
     - `fetch(req, server)`: upgrade to WebSocket, return 200 for non-WS requests
     - `websocket.open(ws)`: subscribe to "planning-state" topic, send full state message with `{ type: "full", state: getFullState(), sequence, timestamp }`
     - `websocket.message(ws, msg)`: handle "refresh" -> send full state
     - `websocket.close(ws)`: unsubscribe from "planning-state"
   - Export `WsServer` type with:
     - `broadcast(diff: StateDiff): void` -- publishes to "planning-state" topic
     - `stop(): void` -- closes the server
     - `getSequence(): number` -- returns current sequence number
   - Maintain a monotonic `sequence` counter. Increment on every broadcast. Include in all messages.
  </action>

  </verify>

1. Create `packages/mission-control/src/server/pipeline.ts`:
   - Export `startPipeline(options: { planningDir: string, wsPort: number }): Promise<PipelineHandle>`
   - On start:
     a. Call `buildFullState(planningDir)` to get initial state
     b. Create WsServer on wsPort with getFullState returning current state
     c. Create file watcher on planningDir with onChange callback
   - onChange callback:
     a. Call `buildFullState(planningDir)` for new state
     b. Call `computeDiff(currentState, newState)`
     c. If diff is not null, update currentState, call wsServer.broadcast(diff)
     d. Wrap in try/catch -- log parse errors, do NOT update state on failure (per research pitfall 1)
   - Set up 5-second reconciliation interval (per research pitfall 2):
     a. Call `buildFullState(planningDir)` for fresh state
     b. Call `computeDiff(currentState, freshState)`
     c. If drift detected, log warning and broadcast
   - Return `PipelineHandle` with `stop()` method that closes watcher, ws server, and clears interval
