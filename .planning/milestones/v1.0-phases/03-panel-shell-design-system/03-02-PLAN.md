1. **PanelSkeleton.tsx** -- Skeleton loading state that varies by panel name. Accept a `variant` prop (string matching panel names: "Sidebar", "Milestone", "Slice Detail", "Active Task", "Chat"). Each variant renders a different skeleton shape reflecting the expected content layout:
   - Sidebar: 1 logo placeholder (h-8 w-8 rounded), 4 nav item rows (h-4 w-full)
   - Milestone: 1 header row (h-4 w-32 + h-2 flex-1 progress bar shape), 4 phase rows (h-3 w-3 circle + h-3 w-16 + h-2 flex-1)
   - Slice Detail: 3 bar chart placeholder rows (varying widths), 2 list blocks
   - Active Task: 1 status dot (h-4 w-4 rounded-full), header text, 3 list items
   - Chat: 3 message bubbles (alternating alignment), 1 input bar at bottom

   Use shadcn/ui `Skeleton` component with `bg-navy-700` class. Follow 8-point spacing grid: use p-4 (16px), gap-4 (16px), p-2 (8px) for dense areas. AVOID p-3, p-5.

2. **PanelEmpty.tsx** -- Empty state with customizable icon, title, and description. Props: `{ icon?: React.ReactNode, title: string, description: string }`. Center vertically and horizontally. Icon from lucide-react default to `Inbox`. Title uses `font-display text-sm font-bold uppercase tracking-wider text-slate-400`. Description uses `font-mono text-xs text-slate-500`. Max width on description: max-w-48.

3. **PanelError.tsx** -- Error state with retry. Props: `{ error: Error, onRetry?: () => void }`. Uses `AlertTriangle` icon from lucide-react in `text-status-error`. Title "Something went wrong" in font-display. Error message in font-mono text-xs. Retry button: `bg-navy-700 px-4 py-2 font-mono text-xs text-cyan-accent hover:bg-navy-600 rounded`. Only show retry button if onRetry provided.

4. **layout-storage.ts** -- Custom PanelGroupStorage adapter. Import `PanelGroupStorage` type from `react-resizable-panels`. Export `createSessionStorage()` function that returns a storage object with:
   - `getItem(name)`: read from localStorage
   - `setItem(name, value)`: write to localStorage (session file sync deferred to Phase 9 per SERV-07)

   This is intentionally simple -- localStorage handles persistence for now. The session file bridge is added in Phase 9 when SERV-07 is implemented.
