# Phase 06.3: New Capabilities — Multi-session Chat, Git Worktree, Settings, Assets - Context

**Gathered:** 2026-03-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Five new capabilities that extend Mission Control beyond single-session chat into a multi-terminal, configurable workspace:
1. Multiple chat sessions per project (up to 4) with tab-based UI
2. Git worktree isolation per session with auto-branching
3. Settings panel for Claude Code configuration and plugin marketplace
4. Project assets storage with gallery view and chat integration
5. Split milestone visualization showing per-session worktree state

</domain>

<decisions>
## Implementation Decisions

### Multi-session Chat Architecture
- Up to 4 simultaneous chat sessions per project
- Each session gets its own ClaudeProcessManager instance (independent Claude subprocess)
- All sessions share the same project's `.planning/` file watcher and state deriver
- Sessions appear as horizontal tabs above the chat area (like VS Code terminal tabs)
- "+" button creates new session with prompt: "Fresh session" or "Fork from current"
- Forked sessions start with the same `--resume` session ID as the source session
- Persist session metadata only (IDs, names, tab order) — not full chat history. Claude `--resume` handles conversation continuity server-side
- Pulsing dot on tab indicates active Claude processing for that session
- Close tab = kill Claude process immediately (no grace period)

### Session Naming
- Sessions start as "Chat 1", "Chat 2", etc. (temporary)
- After first message + response, auto-derive semantic name from message content (like Claude Desktop)
- Tab name, worktree directory, and git branch all match the semantic name
- Double-click tab to rename manually — worktree dir and branch rename to match
- Slug format for filesystem: kebab-case derived from semantic name

### Git Worktree Per Session
- Project-level toggle in settings: "Enable worktree isolation"
- When ON: all new sessions auto-create worktrees at `<project>/.worktrees/<session-slug>/`
- When OFF: sessions share the main working directory
- Add `.worktrees/` to `.gitignore` automatically when first worktree is created
- Each worktree gets auto-created branch: `session/<session-slug>` based off current HEAD
- On session close, prompt user: "Merge changes to main", "Keep worktree for later", or "Delete worktree and branch"
- Each worktree gets its own file watcher feeding state into a split milestone visualization

### Split Milestone Visualization
- When multiple sessions are active with worktrees, milestone view splits to show per-session state
- Each section shows which worktree's `.planning/` changes are being reflected
- Falls back to single view when only one session or worktrees disabled

### Settings Panel
- Accessed via gear icon at bottom of sidebar (below project list, above nothing)
- Opens as full-width view in main area (new ViewType: "settings")
- Changes require explicit "Apply" button (not live)
- Two-tier config: global defaults (`~/.gsd/defaults.json`) with project overrides (`.planning/config.json`)
- Settings panel shows both tiers with "(project override)" indicators

**Configurable sections:**
1. **Claude Code Options**: Permission mode toggle (skip-permissions on/off), model selection, allowed tools whitelist
2. **Skills & Commands**: Browse installed GSD skills and custom slash commands. Enable/disable individual skills
3. **Sub-agents & Profiles**: View/configure GSD model profiles (quality/balanced/budget), see available agents
4. **Worktree Isolation**: Project-level toggle for worktree-per-session
5. **Marketplace**: Hybrid local + npm search — browse installed plugins, search npm for new ones, one-click install (runs npx/npm in background, prompts restart to pick up new commands/skills), delete installed plugins

### Project Assets
- Stored in `.planning/assets/` (inside existing watcher path)
- Upload via drag-drop into chat panel OR dedicated Assets view
- Assets view: new ViewType in sidebar, renders as thumbnail grid gallery
- Supported types: images (PNG, JPG, SVG, GIF), video (MP4, WEBM), documents (PDF, MD, TXT)
- Grid gallery: thumbnails for images/video, file icons for docs. Click to preview full-size. Filter by type
- Upload button in assets view for non-drag-drop workflow
- Toggle between grid and list view (grid default)

### Chat Asset Integration
- Files dragged into chat are stored in `.planning/assets/` and referenced by path
- Chat message shows inline thumbnail preview of the attached asset
- Claude receives the file path in context so it can reference the asset
- Assets view shows all assets regardless of how they were uploaded (chat or direct)

### Claude's Discretion
- WebSocket protocol extensions for multi-session routing (session ID per message)
- How to multiplex multiple ClaudeProcessManager event streams to correct client tabs
- Split milestone view layout details (side-by-side vs stacked)
- Asset file size limits and thumbnail generation approach
- npm search API integration details for marketplace
- Worktree creation/cleanup error handling

</decisions>

<specifics>
## Specific Ideas

- Session naming should work like Claude Desktop — auto-derives name from first message content, not generic numbering
- Tab + worktree + branch must always match in name (single source of truth)
- Milestone view should adapt to show parallel work when multiple sessions have worktrees — like seeing two developers' progress simultaneously
- Marketplace should feel like VS Code extensions panel — search, install, manage from one place
- Assets in chat should show inline thumbnails, not just file links — visual context matters

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `ClaudeProcessManager` (claude-process.ts): Currently singleton per project. Needs to become multi-instance — one per session. Core spawn/event/kill logic is reusable.
- `useChat` hook (useChat.ts): Manages single session state. Needs refactoring to support multiple session states (messages, isProcessing per session).
- `ViewType` (view-types.ts): Currently `chat | milestone | history`. Needs `settings` and `assets` variants.
- `discoverCustomCommands` (discover-commands.ts): Scans filesystem for commands. Reusable for marketplace "installed" view.
- `createFileWatcher` (watcher.ts): Single directory watcher. Needs to support additional watchers for worktree `.planning/` dirs.
- `buildFullState` / `computeDiff` (state-deriver.ts, differ.ts): State pipeline per planning dir. Reusable per worktree.
- `FolderPickerModal` (FolderPickerModal.tsx): Custom modal pattern reusable for session creation prompt, worktree close prompt.

### Established Patterns
- Node `child_process.spawn` for Windows reliability (not Bun.spawn)
- CLAUDECODE env stripping to avoid nested session rejection
- Topic-based pub/sub on WebSocket (`planning-state`, `chat` topics) — needs session-scoped topics
- Route dispatcher pattern (fs-api.ts) for new settings/assets REST endpoints
- `createSwitchGuard` for concurrency protection — reusable for multi-session guards
- ViewType discriminated union with SingleColumnView router — extend for new views

### Integration Points
- `pipeline.ts`: Currently manages single ClaudeProcessManager + single activeClient. Core refactoring point for multi-session.
- `ws-server.ts`: WebSocket server needs session-aware message routing
- `Sidebar.tsx`: Needs gear icon at bottom for settings access
- `ChatView.tsx` / `ChatPanel.tsx`: Need tab bar above chat area and per-session state management
- `AppShell.tsx`: ViewType routing — add settings and assets views
- `MilestoneView.tsx`: Needs split view capability for multi-worktree visualization
- `server.ts`: New REST endpoints for assets upload, settings CRUD, marketplace search/install

</code_context>
