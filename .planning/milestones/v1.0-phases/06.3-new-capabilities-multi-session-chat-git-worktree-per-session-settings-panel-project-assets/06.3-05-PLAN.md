  <files>packages/mission-control/src/lib/view-types.ts, packages/mission-control/src/hooks/useSettings.ts, packages/mission-control/src/hooks/useAssets.ts, packages/mission-control/src/components/chat/ChatDragDropUpload.tsx, packages/mission-control/src/components/chat/ChatAssetThumbnail.tsx</files>
  <action>
    1. Update `view-types.ts`:
       - Add `| { kind: "settings" }` and `| { kind: "assets" }` to ViewType union

    2. Create `useSettings.ts`:
       - Export `SettingsData` type: `{ global: Record<string, unknown>; project: Record<string, unknown>; merged: Record<string, unknown> }`
       - Export `useSettings(apiBase = "http://localhost:4000"): { settings: SettingsData | null; loading: boolean; error: string | null; dirty: boolean; update: (key: string, value: unknown) => void; save: (tier: "global" | "project") => Promise<void>; reload: () => Promise<void> }`
       - On mount: fetch `GET /api/settings`, populate state
       - `update(key, value)`: updates local pending changes (sets dirty=true), does NOT save
       - `save(tier)`: PUT to `/api/settings` with pending changes for given tier, reloads after save, resets dirty
       - `reload()`: re-fetches from server
       - Track pending changes separately from server state

    3. Create `useAssets.ts`:
       - Export `AssetItem` type: `{ name: string; path: string; type: string; size: number; createdAt: number }`
       - Export `useAssets(apiBase = "http://localhost:4000"): { assets: AssetItem[]; loading: boolean; error: string | null; upload: (file: File) => Promise<AssetItem>; remove: (name: string) => Promise<void>; reload: () => Promise<void>; filter: string; setFilter: (f: string) => void; viewMode: "grid" | "list"; setViewMode: (m) => void }`
       - On mount: fetch `GET /api/assets/list`
       - `upload(file)`: POST FormData to `/api/assets/upload`, then reload, returns the uploaded asset metadata
       - `remove(name)`: DELETE `/api/assets/${name}`, then reload
       - `filter`: state for type filtering ("all", "images", "video", "documents")
       - `viewMode`: state for grid/list toggle
       - Computed `filteredAssets` based on filter

    4. Create `ChatDragDropUpload.tsx`:
       - Wrapper component that adds drag-drop file upload capability to the chat panel
       - Props: `onAssetUploaded: (asset: AssetItem) => void; apiBase?: string; children: React.ReactNode`
       - Renders `children` wrapped in a `div` with `onDragOver`, `onDragLeave`, `onDrop` handlers
       - On drag over: show translucent overlay with dashed border and "Drop file to attach" text (bg-cyan-accent/10, border-cyan-accent)
       - On drop: extract files from DataTransfer, call `POST /api/assets/upload` for each file (reuse same FormData pattern as useAssets), call `onAssetUploaded` with returned metadata
       - On drag leave: hide overlay
       - Prevents default browser file handling

    5. Create `ChatAssetThumbnail.tsx`:
       - Inline component for rendering asset thumbnails within chat messages
       - Props: `assetPath: string; assetName: string; assetType: string`
       - For images (png, jpg, jpeg, gif, svg): render `<img>` with `max-w-[200px] max-h-[150px] rounded-md object-cover cursor-pointer`
       - For video (mp4, webm): render `<video>` with play icon overlay, same size constraints
       - For documents (pdf, md, txt): render file icon (lucide FileText) with filename text
       - Image src uses `/api/assets/file/<name>` (server serves the file — add this simple static file route in assets-api if not present, or use a data URL)
       - Click handler opens full preview (same as AssetsView preview)
  </action>
