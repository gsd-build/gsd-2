  <name>Task 1: Create useSessionManager hook with per-session message demuxing and tests</name>
  <files>packages/mission-control/src/hooks/useSessionManager.ts, packages/mission-control/src/hooks/useChat.ts, packages/mission-control/tests/useSessionManager.test.ts</files>
  <behavior>
    - selectSession switches activeSessionId, activeMessages reflects that session's messages
    - createSession sends session_create over WebSocket
    - closeSession sends session_close; if closing active session, auto-switches to first remaining
    - renameSession sends session_rename over WebSocket
    - sendMessage includes activeSessionId in the chat payload
    - session_update WS messages update the sessions list
    - chat_event/chat_complete with sessionId routes to correct session's message buffer
    - Messages for inactive sessions accumulate silently (visible when tab is selected)
    - isActiveProcessing reflects only the active session's processing state
    - project_switched clears all sessions and requests fresh session_list
  </behavior>
  <action>
    1. Create `useSessionManager.ts`:
       - Export `SessionTab` interface: `{ id: string; name: string; isProcessing: boolean; hasWorktree: boolean }`
       - Export `UseSessionManagerResult`: `{ sessions: SessionTab[]; activeSessionId: string; activeMessages: ChatMessage[]; isActiveProcessing: boolean; selectSession: (id: string) => void; createSession: (forkFrom?: string) => void; closeSession: (id: string) => void; renameSession: (id: string, name: string) => void; sendMessage: (text: string) => void; permissionPrompt: PermissionPromptEvent | null; respondToPermission: (action) => void }`
       - State: `sessions` (SessionTab[]), `activeSessionId` (string), `messagesBySession` (Map<string, ChatMessage[]>), `processingBySession` (Map<string, boolean>)
       - Share a SINGLE WebSocket connection via `useReconnectingWebSocket`
       - On WebSocket message:
         - `session_update`: update sessions list from server metadata
         - `chat_event` / `chat_complete` / `chat_error`: route to correct session using `msg.sessionId`, update `messagesBySession` and `processingBySession` for that session
         - Apply same stream processing logic as existing useChat (`processStreamEvent`)
         - `permission_prompt`: set permissionPrompt state
         - `project_switched`: clear all sessions, request fresh session_list
       - `sendMessage(text)`: send `{ type: "chat", prompt: text, sessionId: activeSessionId }`
       - `createSession(forkFrom?)`: send `{ type: "session_create", forkFromSessionId: forkFrom }`
       - `closeSession(id)`: send `{ type: "session_close", sessionId: id }`. If closing active session, switch to first remaining.
       - `renameSession(id, name)`: send `{ type: "session_rename", sessionId: id, name }`
       - On mount: send `{ type: "session_list" }` to get initial sessions
       - Expose `activeMessages` as `messagesBySession.get(activeSessionId) ?? []`
       - Expose `isActiveProcessing` as `processingBySession.get(activeSessionId) ?? false`

    2. Refactor `useChat.ts`:
       - Keep `processStreamEvent` exported (it's used by useSessionManager)
       - Keep the existing `useChat` hook intact for backward compat but mark with `@deprecated` JSDoc comment — useSessionManager replaces it
       - No breaking changes to useChat

    3. Write tests in `tests/useSessionManager.test.ts`:
       - Test message routing: chat_event with sessionId A goes to session A's buffer, not session B's
       - Test selectSession switches activeMessages to selected session's buffer
       - Test closeSession on active session auto-switches to remaining session
       - Test sendMessage includes correct sessionId in payload
       - Test session_update updates the sessions list
       - Test isActiveProcessing reflects only active session
       - Mock WebSocket with simple send/receive stub
       - Use `@testing-library/react` renderHook for hook testing
  </action>
