  <name>Task 1: Git worktree CRUD operations with .gitignore management</name>
  <files>packages/mission-control/src/server/worktree-api.ts, packages/mission-control/tests/worktree-api.test.ts</files>
  <behavior>
    - createSessionWorktree creates worktree at repoRoot/.worktrees/slug/ with branch session/slug
    - createSessionWorktree returns { worktreePath, branchName } on success
    - createSessionWorktree returns { error } on git failure
    - createSessionWorktree calls ensureGitignoreEntry before creating worktree
    - ensureGitignoreEntry appends ".worktrees/" to .gitignore if not already present
    - ensureGitignoreEntry creates .gitignore if it doesn't exist
    - ensureGitignoreEntry is idempotent (no duplicate entries)
    - removeSessionWorktree removes worktree and optionally deletes the branch
    - removeSessionWorktree returns { ok } on success, { error } on failure
    - listWorktrees returns array of { path, branch, head } from git worktree list
    - All paths are forward-slash normalized
  </behavior>
  <action>
    1. Create `worktree-api.ts`:
       - Import `spawn as nodeSpawn` from `node:child_process`, `readFile`, `writeFile`, `access` from `node:fs/promises`, `join` from `node:path`
       - Helper `spawnGit(cwd: string, args: string[]): Promise<{ ok: boolean; stdout: string; stderr: string }>` â€” wraps nodeSpawn in Promise, collects stdout/stderr, resolves on close. Follows exact pattern from git-api.ts.
       - `ensureGitignoreEntry(repoRoot: string, entry: string): Promise<void>` â€” reads `.gitignore`, checks if entry line exists (trimmed comparison), appends with newline if not. Creates file if not found.
       - `createSessionWorktree(repoRoot: string, sessionSlug: string): Promise<{ worktreePath: string; branchName: string } | { error: string }>`:
         - Compute worktreePath = `join(repoRoot, ".worktrees", sessionSlug).replace(/\\\\/g, "/")`
         - Compute branchName = `session/${sessionSlug}`
         - Call `ensureGitignoreEntry(repoRoot, ".worktrees/")`
         - Call `spawnGit(repoRoot, ["worktree", "add", "-b", branchName, worktreePath])`
         - Return success or error based on result
       - `removeSessionWorktree(repoRoot: string, worktreePath: string, deleteBranch: boolean = false): Promise<{ ok: boolean; error?: string }>`:
         - Normalize worktreePath to forward slashes
         - Call `spawnGit(repoRoot, ["worktree", "remove", worktreePath, "--force"])`
         - If deleteBranch, derive branch name from worktree path slug, call `spawnGit(repoRoot, ["branch", "-D", branchName])`
         - Return result
       - `listWorktrees(repoRoot: string): Promise<Array<{ path: string; branch: string; head: string }>>`:
         - Call `spawnGit(repoRoot, ["worktree", "list", "--porcelain"])`
         - Parse porcelain output (blocks separated by blank lines, each with "worktree", "HEAD", "branch" lines)
         - Return parsed array

    2. Write tests in `tests/worktree-api.test.ts`:
       - `ensureGitignoreEntry` tests: use temp directory, test creates file, test appends entry, test idempotency
       - `spawnGit` helper can be tested indirectly via `listWorktrees` if inside a git repo
       - For `createSessionWorktree` / `removeSessionWorktree`: these require a real git repo. Create a temp git repo in beforeAll (`git init`), test create then list then remove cycle.
       - Test path normalization (forward slashes)
       - Mark worktree create/remove tests with a comment noting they need real git â€” if CI has no git, they can be skipped
  </action>
