    1. Add `renameSessionWorktree` to `worktree-api.ts`:
       - Export `renameSessionWorktree(repoRoot: string, oldSlug: string, newSlug: string): Promise<{ worktreePath: string; branchName: string } | { error: string }>`
       - Implementation:
         a. Remove the old worktree: `spawnGit(repoRoot, ["worktree", "remove", oldWorktreePath, "--force"])`
         b. Rename the branch: `spawnGit(repoRoot, ["branch", "-m", oldBranchName, newBranchName])`
         c. Re-add worktree at new path with existing branch: `spawnGit(repoRoot, ["worktree", "add", newWorktreePath, newBranchName])`
         d. Return new { worktreePath, branchName } on success, { error } on failure
       - Old path: `.worktrees/<oldSlug>`, new path: `.worktrees/<newSlug>`
       - Old branch: `session/<oldSlug>`, new branch: `session/<newSlug>`

    2. Update `SessionManager` in `session-manager.ts`:
       - Add `worktreeEnabled: boolean` property (read from settings on construction, toggled via method)
       - Add `setWorktreeEnabled(enabled: boolean): void` method
       - In `createSession()`: if `worktreeEnabled`, import and call `createSessionWorktree(repoRoot, session.slug)`. Set `session.worktreePath` and `session.worktreeBranch`. If worktree creation fails, log error but still create session (graceful degradation). Pass worktreePath as cwd to ClaudeProcessManager when worktree active.
       - Add `async closeSessionWithWorktree(id: string, action: "merge" | "keep" | "delete"): Promise<void>`:
         - "merge": TODO marker for `git merge session/<slug>` (complex, leave as future work with clear comment), then remove worktree and branch
         - "keep": just remove session from registry, leave worktree on disk
         - "delete": call `removeSessionWorktree(repoRoot, worktreePath, true)`, then remove from registry
       - In `renameSession()`: Per user decision, rename the worktree directory and git branch to match the new name. Call `renameSessionWorktree(repoRoot, oldSlug, newSlug)`. Update `session.worktreePath` and `session.worktreeBranch` to new values. If rename fails (e.g., git error), log warning but still update display name — the worktree keeps its old path as fallback. Update the ClaudeProcessManager cwd to the new worktree path if rename succeeded.

    3. Add auto-naming in `useSessionManager.ts`:
       - After first `chat_complete` event for a session that still has default name ("Chat N"):
         - Extract first user message content for that session
         - Derive name: take first 30 chars, clean to title case, trim trailing incomplete word
         - Send `session_rename` with derived name
         - This matches "like Claude Desktop" user decision
         - The server-side renameSession will handle renaming the worktree dir and branch to match

    4. Update `pipeline.ts`:
       - Wire worktree close actions: add `onSessionClose` handler in ws-server options that receives close action (merge/keep/delete) and calls `sessionManager.closeSessionWithWorktree`
       - Read worktree-enabled setting from project config on startup and on settings change

    5. Update `MilestoneView.tsx`:
       - Accept optional `sessions` prop: `Array<{ id: string; name: string; worktreePath: string | null }>`
       - When multiple sessions have worktrees (>1 with non-null worktreePath):
         - Show a tabbed or stacked split view with one section per worktree session
         - Each section header: session name + branch name badge
         - Each section renders the standard milestone content but filtered to that worktree's state
       - When only one session or no worktrees: render standard single milestone view (no change)
       - Use `react-resizable-panels` (already in project) for side-by-side split if 2 sessions, stacked if 3-4
       - Fallback gracefully: if worktree state is unavailable, show "Watching main branch" label
