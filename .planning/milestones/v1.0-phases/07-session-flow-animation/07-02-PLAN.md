  <behavior>
    - useSessionFlow returns "initializing" when wsStatus is not "connected"
    - useSessionFlow returns "onboarding" when state is null or has empty phases AND empty roadmap.phases
    - useSessionFlow returns "resume" when state has data AND /api/session/status returns continueHere
    - useSessionFlow returns "dashboard" when state has data AND no continue-here
    - dismissResume transitions from "resume" to "dashboard"
    - dismiss transitions from "onboarding" to "dashboard"
  </behavior>
  <action>
    1. Create `src/hooks/useSessionFlow.ts`:
       - Export type `SessionMode = "initializing" | "onboarding" | "resume" | "dashboard"`
       - Export interface `ContinueHereData { phase: string; task: number; totalTasks: number; status: string; currentState: string; nextAction: string; }`
       - Export function `useSessionFlow(state: PlanningState | null, wsStatus: ConnectionStatus)`
       - Returns `{ mode: SessionMode; continueHere: ContinueHereData | null; dismiss: () => void }`
       - Logic:
         a) If wsStatus !== "connected" -> mode = "initializing"
         b) If state is null OR (state.phases.length === 0 AND state.roadmap.phases.length === 0) -> mode = "onboarding"
         c) Else fetch /api/session/status once (useEffect with state dependency, guarded by ref to avoid refetch). If continueHere -> mode = "resume". Else -> mode = "dashboard"
         d) `dismiss()` sets mode to "dashboard" and clears continueHere (works for both onboarding and resume)
       - Use useRef flag to prevent re-fetching session status on every state update

    2. Create `tests/session-flow.test.tsx` with hook tests:
       - Test useSessionFlow returns "initializing" when wsStatus is "connecting"
       - Test useSessionFlow returns "onboarding" when state is null
       - Test useSessionFlow returns "onboarding" when state has empty phases and roadmap
       - Test useSessionFlow returns "dashboard" when state has data and no continue-here (mock fetch)
       - Test useSessionFlow returns "resume" when state has data and continue-here exists (mock fetch)
       - Test dismiss transitions from "resume" to "dashboard"
  </action>
