## Known Gaps (Deferred to Phase 6.2)

The following issues were identified during human verification and are NOT bugs in this plan's scope. They represent integration gaps that require deeper work across multiple subsystems:

1. **Panel state refresh after project switch:** After clicking "Open Here" on a non-GSD folder, the right-side dashboard panels do not update to reflect the newly opened folder. The pipeline switch broadcasts state but panel components may not be re-rendering on the new state.

2. **Claude Code sub-agent streaming:** Only the main Claude Code process streams to the dashboard. Sub-agent processes spawned by Claude Code do not have their output captured or streamed.

3. **Dashboard feels static:** The dashboard needs live data streaming from Claude Code execution to feel responsive. Currently only file-watcher-based state updates flow through.

4. **Claude Code --dangerously-skip-permissions flag:** The Claude Code process should be launched with this flag for autonomous operation within Mission Control.

5. **Sidebar nav stubs:** Activity, Verify, and History nav items in the sidebar are visual stubs with no functionality wired up.

These items are tracked for a new Phase 6.2 focused on streaming infrastructure and dashboard liveness.
