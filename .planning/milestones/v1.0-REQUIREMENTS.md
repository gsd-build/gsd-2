# Requirements: GSD Mission Control

**Defined:** 2026-03-10
**Core Value:** A developer types in Mission Control's chat, Claude Code executes, code lands, and dashboard panels update in real time — the full build loop in one window.

## v1 Requirements

### Server Infrastructure

- [x] **SERV-01**: Bun server starts with single `bun run dev` command, serving dashboard on :4000
- [x] **SERV-02**: File watcher monitors `.planning/` recursively, detecting all file changes
- [x] **SERV-03**: State derivation engine parses `.planning/` files into structured state objects
- [x] **SERV-04**: WebSocket server on :4001 pushes diff-only state updates to all connected clients
- [x] **SERV-05**: File event to panel update completes under 100ms
- [x] **SERV-06**: Dev server proxy routes `/preview` path to configurable project dev server port
- [x] **SERV-07**: Session file (`.planning/.mission-control-session.json`) persists layout prefs, chat history (last 50), last viewed state
- [x] **SERV-08**: WebSocket reconnects with exponential backoff on disconnect
- [x] **SERV-09**: Bun process restart reconstructs full state from `.planning/` files

### Panel Layout

- [x] **PNLS-01**: Five-panel layout renders: Sidebar (200px), Milestone View (flex), Slice Detail (280px), Active Task (300px), Chat (340px)
- [x] **PNLS-02**: All panels are resizable via drag handles using react-resizable-panels
- [x] **PNLS-03**: Panel layout preferences persist in session file across restarts
- [x] **PNLS-04**: Every panel has designed loading, empty, and error states (skeleton screens, not spinners)
- [x] **PNLS-05**: Design system implemented: dark navy base, cyan accent for active/CTAs only, 60/30/10 color rule
- [x] **PNLS-06**: Typography: Share Tech Mono for headers, JetBrains Mono for data. 4 sizes (10/12/14/18px), 2 weights (400/700)
- [x] **PNLS-07**: 8-point spacing grid enforced across all padding, margin, and gap values

### Sidebar Panel

- [x] **SIDE-01**: GSD pixel-art logo rendered as SVG matching terminal screenshot
- [x] **SIDE-02**: Project list shows active/paused status per project
- [x] **SIDE-03**: Navigation items: Projects, Activity, Verify, History
- [x] **SIDE-04**: Claude Code connection indicator: pulsing cyan dot with ACTIVE/DISCONNECTED label
- [x] **SIDE-05**: Current model profile displayed from `.planning/config.json`

### Milestone View Panel

- [x] **MLST-01**: Header shows git branch, milestone name (Share Tech Mono), overall progress bar, tasks complete/total
- [x] **MLST-02**: Phase list rows show status icon, phase ID in cyan, task progress bar, demo sentence
- [x] **MLST-03**: Completed phases show their squash commit message
- [x] **MLST-04**: Committed history section shows all squash merge commits at panel bottom

### Slice Detail Panel

- [x] **SLCD-01**: Context usage bar chart with one bar per task, color-coded by budget (green <50%, amber 50-70%, red >70%)
- [x] **SLCD-02**: Boundary map shows PRODUCES (green-bordered) and CONSUMES (blue-bordered) lists
- [x] **SLCD-03**: UAT status rows per completed phase with test count, verification bars, and status badges

### Active Task Panel

- [x] **TASK-01**: Executing state: pulsing amber dot, task ID, wave number, context budget meter with color shift
- [x] **TASK-02**: Must-haves list with completion state, tier badges (BEHAVIORAL, STATIC, COMMAND, HUMAN)
- [x] **TASK-03**: Target files list with FileCode icons
- [x] **TASK-04**: Checkpoint reference showing git checkpoint before task started
- [x] **TASK-05**: Waiting state: last completed summary, next task name, run-next-task prompt

### Chat Panel

- [x] **CHAT-01**: Chat input at bottom with `/` prefix hint and autocomplete for GSD slash commands
- [x] **CHAT-02**: Claude Code spawned as child process from Bun with piped stdout streaming
- [x] **CHAT-03**: Streaming responses render token-by-token in real time
- [x] **CHAT-04**: Agent responses visually distinguished from system messages
- [x] **CHAT-05**: State panels animate as files land on disk during execution
- [x] **CHAT-06**: Command history recalled with up arrow
- [x] **CHAT-07**: Chat routing under 200ms excluding model latency

### File System Integration

- [x] **FS-01**: Directory listing API returns entries with name, path, isDirectory, and isGsdProject fields
- [x] **FS-02**: GSD project detection identifies directories containing `.planning/` subdirectory
- [x] **FS-03**: Path traversal prevention rejects `..` segments and returns 400; permission errors return 403
- [x] **FS-04**: Pipeline hot-switch reconfigures watcher, state deriver, and Claude process for new project directory
- [x] **FS-05**: Recent projects persisted to `~/.gsd/recent-projects.json`, deduped, trimmed to 20 entries
- [x] **FS-06**: Interactive file tree browser in sidebar with lazy-loading expand/collapse and GSD badges
- [x] **FS-07**: Project switching from UI triggers full pipeline reconfiguration and state broadcast

### Live Streaming + Native OS Integration

- [x] **LS-01**: Claude Code process streams all events (tool_use, text, thinking, result) to dashboard via WebSocket in real-time
- [x] **LS-02**: Sub-agent/Task tool spawned processes also stream their events to dashboard (not just main process)
- [x] **LS-03**: Claude Code process launches with --dangerously-skip-permissions flag by default (with option to disable)
- [x] **LS-04**: "Open Folder" triggers native OS file dialog (like VS Code) instead of in-panel tree browser
- [x] **LS-05**: After opening/switching project, entire dashboard state reloads (WebSocket reconnects to new planning dir, all panels refresh)
- [x] **LS-06**: Sidebar Activity nav item shows live Claude Code stream activity (tool calls, file writes, thinking indicators)
- [x] **LS-07**: Sidebar Verify nav item links to /gsd:verify-work functionality
- [x] **LS-08**: Sidebar History nav item shows committed history / git log for current project
- [x] **LS-09**: Permission prompts from Claude Code stream to dashboard for user approval (even with --dangerously-skip-permissions as fallback)

### Multi-Session Chat

- [x] **MULTI-01**: Up to 4 simultaneous chat sessions per project, each with its own Claude Code process
- [x] **MULTI-02**: Session tabs above chat area with processing indicators, inline rename, and close actions
- [x] **MULTI-03**: Session-scoped WebSocket message routing with sessionId demux, max 4 sessions enforced

### Git Worktree Per Session

- [x] **WKTREE-01**: Git worktree CRUD operations (create, remove, rename, list) with session integration
- [x] **WKTREE-02**: Auto-management of .gitignore for .worktrees/ directory and session/<slug> branch naming

### Settings Panel

- [x] **SETTINGS-01**: Settings panel with two-tier configuration (global ~/.gsd/defaults.json + project .planning/config.json) and explicit Apply
- [x] **SETTINGS-02**: Five collapsible settings sections (Claude Code, Skills, Sub-agents, Worktree, Marketplace) with project override indicators

### Project Assets

- [x] **ASSETS-01**: Asset upload, listing, and deletion via REST API with 50MB size limit and extension validation
- [x] **ASSETS-02**: Thumbnail grid gallery view with grid/list toggle, type filtering, drag-drop upload in chat, and file serving endpoint

### Discuss Mode

- [x] **DISC-01**: Activates during `/gsd:discuss-phase` with distinct visual treatment
- [x] **DISC-02**: Agent questions render as cards with question text prominent
- [x] **DISC-03**: Multiple-choice options render as button groups, not raw text
- [x] **DISC-04**: Free-text questions render as input fields
- [x] **DISC-05**: Progress indicator showing questions remaining
- [x] **DISC-06**: Decision log sidebar showing locked answers

### Review Mode

- [x] **REVW-01**: Activates after `/gsd:ui-review` completes
- [x] **REVW-02**: 6-pillar score table with count-up animation on reveal
- [x] **REVW-03**: Each pillar row expandable to show specific findings
- [x] **REVW-04**: Top 3 priority fixes as action cards with Fix quick-action

### Live Preview

- [x] **PREV-01**: Preview panel toggled via Cmd+P or toolbar button, slides in from right
- [x] **PREV-02**: Viewport switcher: Desktop (1440px), Tablet (768px), Mobile (375px)
- [x] **PREV-03**: Live iframe proxied through Bun to project dev server
- [x] **PREV-04**: Dev server offline shows clean empty state, not broken iframe

### Session Flow

- [x] **SESS-01**: New project (no `.planning/`): onboarding screen with Remotion logo animation, guided `/gsd:new-project` chat
- [x] **SESS-02**: Existing project: dashboard loads current state, resume card if continue-here file exists
- [x] **SESS-03**: Project selector for multiple GSD projects sorted by most recent activity

### Animation

- [x] **ANIM-01**: Remotion logo animation: pixel-art build 600ms, plays once on session start

- [x] **ANIM-05**: Chat message arrival: slide-up 80ms, opacity 0->1

- [x] **KEYS-01**: Command palette via Ctrl+Shift+P with GSD command search
- [x] **KEYS-02**: Panel focus switching via keyboard shortcuts
- [x] **KEYS-03**: One h1 per view, logical heading hierarchy throughout
- [x] **KEYS-04**: All interactive elements have accessible names
- [x] **KEYS-05**: Minimum 44px touch targets on all interactive elements
- [x] **KEYS-06**: Focus managed on panel transitions

- [x] **MONO-01**: Mission Control lives in `packages/mission-control/` workspace
- [x] **MONO-02**: Bun workspace configuration with workspace:* protocol
- [x] **MONO-03**: GSD core remains at repo root, publishes independently

## Out of Scope

| Feature | Reason |
|---------|--------|
| Direct code editing | Mission Control is a command center, not an IDE. All mutations through Claude Code or gsd-tools. |
| Cloud sync / collaboration | Violates zero-external-services constraint. Git handles sharing naturally. |
| Plugin/extension system | Extension APIs expensive to design well. Ship opinionated defaults first. |
| Telemetry | Requires data collection infrastructure and privacy considerations. |
| Database | All state derived from `.planning/` files. No persistence layer needed. |
| Node.js / Vite | Bun is sole runtime. No fallback to Node tooling. |
