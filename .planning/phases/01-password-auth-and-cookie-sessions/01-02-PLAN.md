Create the auth API endpoints (login + logout + status) and extend proxy.ts with cookie-based session verification.

Purpose: These endpoints own the session cookie lifecycle. The status endpoint lets the login gate check whether auth is configured and the current session is valid. The proxy extension enables cookie auth for Tailscale HTTPS users while preserving the existing bearer token flow for localhost users.
Output: Three new API routes and an updated proxy.ts that checks cookies before bearer tokens.
</objective>

    4. Add the Edge-compatible verification function at the top of the file (after imports). This uses `crypto.subtle` (Web Crypto API, available in Edge Runtime) to verify the custom HMAC token format from `web-session-auth.ts`:
       ```typescript
       async function verifySessionCookieEdge(token: string): Promise<boolean> {
         const secret = process.env.GSD_WEB_SESSION_SECRET
         if (!secret) return false

         const dotIndex = token.indexOf(".")
         if (dotIndex === -1) return false

         const payloadB64 = token.slice(0, dotIndex)
         const signatureHex = token.slice(dotIndex + 1)

         try {
           const encoder = new TextEncoder()
           const key = await crypto.subtle.importKey(
             "raw",
             encoder.encode(secret),
             { name: "HMAC", hash: "SHA-256" },
             false,
             ["sign"]
           )
           const signatureBytes = await crypto.subtle.sign("HMAC", key, encoder.encode(payloadB64))
           const expectedHex = Array.from(new Uint8Array(signatureBytes))
             .map(b => b.toString(16).padStart(2, "0"))
             .join("")

           // Constant-time comparison (AUTH-10)
           if (signatureHex.length !== expectedHex.length) return false
           let mismatch = 0
           for (let i = 0; i < signatureHex.length; i++) {
             mismatch |= signatureHex.charCodeAt(i) ^ expectedHex.charCodeAt(i)
           }
           if (mismatch !== 0) return false

           // Parse and check expiry
           const payloadJson = atob(payloadB64.replace(/-/g, "+").replace(/_/g, "/"))
           const payload = JSON.parse(payloadJson)
           if (!payload.expiresAt || payload.expiresAt <= Date.now()) return false

           return true
         } catch {
           return false
         }
       }
       ```

    5. The existing bearer token check (lines 54-76 in current proxy.ts) remains EXACTLY as-is. Do not modify the origin check or bearer token logic. The cookie check is purely additive — it short-circuits before the bearer check on success.

    **CRITICAL:** The session secret must be passed to the Edge Runtime via an environment variable `GSD_WEB_SESSION_SECRET`. Plan 01-04 Task 2 injects this into the `web-mode.ts` env block at server startup. If the env var is absent, the cookie path is skipped and bearer token auth remains the only path — this preserves backward compatibility (AUTH-08).

    **Why crypto.subtle instead of jose:** The session token format is `base64url(JSON).hex(HMAC)` (custom format from Plan 01). Using `crypto.subtle` directly avoids adding `jose` as a dependency for this specific verification. If `jose` is already installed for other reasons, it could be used instead, but `crypto.subtle` is built into the Edge Runtime.
