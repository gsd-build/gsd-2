<domain>
## Phase Boundary

Fix 10 identified bugs/regressions in the shipped M2 app and add one new feature (Code Explorer). All work is scoped to `packages/mission-control/`. No new GSD workflow capabilities, no server-side schema changes.

<decisions>
## Implementation Decisions

### Preview panel crash (tablet/mobile/dual)
- Crash type: whole app freezes or reloads — indicates unhandled error in React tree (not just blank iframe)
- Add React error boundary around each DeviceFrame so a single viewport crash doesn't take down the app
- Each device frame must be isolated — one crashing iframe should not affect others

### Dev server detection
- Empty state (when no server detected): show a "Scan for servers" button that checks common ports (3000, 4173, 5173, 8080, 8000) PLUS a manual port number input field as fallback
- Scan checks ports by attempting a fetch to determine if something is listening; filter by content-type to identify frontend vs backend
- When a server is found via scan, set it as the active port automatically

### Multi-server + viewport architecture
- **Desktop tab**: has a sub-tab selector for multiple detected servers — supports BOTH frontend AND backend servers (GSD is full-stack, not just frontend)
- **Tablet + Mobile tabs**: frontend servers only (responsive design check); share the selected frontend server
- **Dual tab**: Expo-style mobile preview — each device frame (left = Android, right = iOS/web) has its OWN independent server selector; supports two different servers running simultaneously (e.g., :8081 and :19006)
- **All viewport iframes**: fully interactive / clickable — behave exactly as if previewing localhost:PORT in Chrome (not sandboxed read-only)
- Shared frontend server: switching frontend server in Desktop tab updates Tablet and Mobile tabs

### Code Explorer
- **Entry point**: `<>` button in the sidebar (both expanded and collapsed states); also accessible as a nav item
- **UI**: Opens as a full-screen modal overlay (not a route/view swap)
  - Left pane: file tree navigation (project root scoped — cannot navigate above project directory)
  - Right pane: file content viewer / editor with syntax highlighting
  - X button top-right to close; clicking backdrop outside modal also closes
  - Modal reopens to last opened file on re-open (persisted in localStorage or session state)
- **File actions** (VSCode-like, scoped to project directory):
  - Click file → open in right pane
  - Edit content inline (Monaco editor or CodeMirror preferred)
  - Save: writes directly to disk immediately via server API (no confirm dialog)
  - Copy path to clipboard (right-click context menu or toolbar button)
  - Open in external editor (VS Code via `code` CLI or Tauri `shell.open`)
  - Folder expand/collapse in tree
- **Scope**: strictly project root and below — boundary enforcer rules apply

### Project name display
- **Sidebar**: the bordered section above New Window already exists in `Sidebar.tsx` — fix the condition so it shows when `state?.projectState` has a name (not just when `activeProjectPath` is set)
- **Right panel**: add a thin top header bar to `SingleColumnView` showing the project name + optional path breadcrumb; visible regardless of active view (Chat, Milestones, Settings, etc.)
- Project name source priority: `activeProjectPath` basename → `state?.projectState?.milestone_name` → hidden

### Logo fix
- Switch `GsdLogo` component from PNG (`gsd-2-mission-control-logo.png`) to SVG version
- If no SVG asset exists: render `GSD` in Share Tech Mono as a styled text element (always sharp, no image asset needed)
- Remove any `className` size overrides that were compensating for PNG blur

### Session fork
- `onCreateSession(forkFrom?: string)` already accepts a session ID — fix the wiring so the fork actually copies message history from the parent session
- Forked session: starts with all messages from the parent session copied in (allows exploring alternative approaches from same context)
- Fork creates a new git worktree as normal; the copied messages are display-only context (not re-sent to Claude)

### Refresh recovery
- After page refresh + WebSocket reconnect: if a session has `processing: true` (yellow dot) but no active stream arrives within 3 seconds, show a non-intrusive banner:
  _"GSD may still be running. Reconnect to attach to the session."_ with a **Reconnect** button
- Clicking Reconnect: re-subscribes to the WebSocket and resumes streaming if the process is still alive; if the process finished, clears the processing state and shows the result
- Auto-reconnect does NOT happen silently — the user must click Reconnect to avoid surprising behavior

### Scrollbar utility
- The existing `.scrollbar-thin` CSS class + Firefox inline styles in `ChatPanel.tsx` must become a reusable pattern
- Extract to a shared utility: either a CSS class in `globals.css` (already exists as `.scrollbar-thin`) or a `cn()` helper constant
- Apply to all scrollable areas: Settings page, Milestones list, Sidebar project tree, any overflow-y-auto containers

### Claude's Discretion
- Exact Monaco vs CodeMirror choice for Code Explorer editor (whichever integrates cleanly with Bun/React 19)
- Port scan implementation detail (parallel fetch race or sequential)
- Exact DeviceFrame dimensions for the dual tab (Expo device sizes)
- Syntax highlighting theme in Code Explorer (should match navy palette)
- Exact breadcrumb format in right panel header

</decisions>
