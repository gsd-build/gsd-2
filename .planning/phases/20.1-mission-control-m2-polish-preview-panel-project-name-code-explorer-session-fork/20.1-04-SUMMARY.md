# Phase 20.1 Plan 04: Code Explorer Summary

**One-liner:** CodeMirror 6 full-screen modal with lazy file tree, language detection (ts/tsx/js/json/md/css/py), and project-root-scoped read/write API.

## What Was Built

### Task 1: CodeMirror 6 + fs read/write API (already committed in 70aff76)

- Installed 9 CodeMirror 6 packages (view, state, basic-setup, 5 language packs, one-dark theme)
- Added `GET /api/fs/read` and `POST /api/fs/write` handlers to `fs-api.ts`
- `handleFsRequest` signature updated to accept `allowedRoot?: string` for boundary enforcement
- `server.ts` passes `repoRoot` to `handleFsRequest`, scoping all file access to the project root
- Created `tests/fs-api-read-write.test.ts` with 6 source-text verification tests

### Task 2: Code Explorer Components + Sidebar Button + Modal Wiring

- **useCodeExplorer.ts** — hook managing `isOpen`, `openExplorer`, `closeExplorer`, `selectedFile`, `selectFile`, `lastFile` (with localStorage persistence)
- **FileTree.tsx** — recursive tree component: loads via `/api/fs/list`, lazy dir expansion with `expandedDirs` Set, Folder/FolderOpen/File lucide icons, selected file highlighted cyan
- **FileEditor.tsx** — CodeMirror 6 wrapper: EditorView recreated per filePath, `basicSetup` + `oneDark`, language detection, `Mod-s` keybinding calls `onSave`
- **CodeExplorer.tsx** — full-screen modal (95vw x 90vh, z-50 overlay): file read via `/api/fs/read`, save via `/api/fs/write`, dirty indicator, copy-path button, X + backdrop click close
- **Sidebar.tsx** — added `onOpenCodeExplorer?: () => void` prop, Code icon button in both expanded and collapsed layouts
- **AppShell.tsx** — wires `useCodeExplorer`, passes `openExplorer` to Sidebar, renders `<CodeExplorer>` modal with `activeProjectPath`
