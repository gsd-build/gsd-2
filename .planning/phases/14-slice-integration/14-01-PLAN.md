    ## Parser behaviors (for RED tests)

    ### parseRoadmap(raw: string): GSD2RoadmapState
    - Extracts milestone ID and name from first `# M001 —` heading or frontmatter
    - Finds slice sections by `## S{NN}` headings
    - For each slice: extracts name, status from badge text (PLANNED/IN PROGRESS/NEEDS REVIEW/COMPLETE), task count from "N tasks", cost from "~$N.NN", branch from "gsd/M001/SNN"
    - Parses "Depends on: SXX Name" lines into dependency list
    - Missing file or no slices → { milestoneId: "M001", milestoneName: "", slices: [] }

    ### parsePlan(raw: string, sliceId: string): GSD2SlicePlan
    - Reads tasks from `## Tasks` section — numbered items `T01: Name [status]`
    - Reads cost estimate from frontmatter `cost_estimate: ~$0.40`
    - Reads must_haves from `## Must-Haves` block
    - Missing file → { sliceId, costEstimate: null, tasks: [], mustHaves: [] }

    ### parseUat(raw: string, sliceId: string): GSD2UatFile
    - Parses `- [ ] UAT-01: Description` and `- [x] UAT-01: Description` lines
    - Returns items with checked=true when [x], checked=false when [ ]
    - Missing file → { sliceId, items: [] }

    ## Test cases
    - parseRoadmap with realistic M001-ROADMAP.md → returns 3 slices with correct statuses
    - parseRoadmap with empty string → returns empty slices array (no throw)
    - parsePlan with realistic S01-PLAN.md → returns tasks with ids and statuses
    - parsePlan with empty string → returns empty tasks array (no throw)
    - parseUat with mixed checked/unchecked → checked field correctly set
    - parseUat with empty string → returns empty items array (no throw)
    - buildFullState with .gsd/ containing M001-ROADMAP.md → state.slices has entries
    - buildFullState with no UAT file → state.uatFile is null (no throw)
  </behavior>
