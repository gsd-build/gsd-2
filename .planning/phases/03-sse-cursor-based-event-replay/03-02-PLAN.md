<objective>
Extend the SSE endpoint to support cursor-based replay with a strict replay-to-live handoff protocol. When a client reconnects with `?since=N`, the endpoint captures a replay ceiling seq, streams missed events up to that ceiling, flushes buffered live events beyond the ceiling, handles stale cursors with a snapshot signal, and sends a `live` sentinel when replay is complete.

Purpose: This is the server-side replay mechanism. Without it, reconnecting browsers would only see events from the moment of reconnection, missing everything that happened while the browser was closed.
Output: Modified `web/app/api/session/events/route.ts`
</objective>

    3. **Parse and strictly validate `since` query parameter** at top of GET handler, after `requireProjectCwd`:
       ```typescript
       const url = new URL(request.url);
       const sinceParam = url.searchParams.get("since");
       let sinceSeq = -1; // Default: no cursor, skip replay
       if (sinceParam !== null) {
         const parsed = parseInt(sinceParam, 10);
         // Only accept non-negative integers; reject NaN, negative, decimal
         if (!isNaN(parsed) && parsed >= 0 && String(parsed) === sinceParam) {
           sinceSeq = parsed;
         }
       }
       const hasReplayCursor = sinceSeq >= 0;
       ```

    4. **Get EventLog reference** from bridge:
       ```typescript
       const eventLog = bridge.getEventLog();
       ```

    5. **Rewrite the `ReadableStream.start()` callback** to implement the replay-ceiling-then-live pattern:

       a. **Subscribe to live events first** (captures events arriving during replay):
       ```typescript
       const liveBuffer: unknown[] = [];
       let isReplaying = hasReplayCursor;
       let bufferOverflow = false;

       unsubscribe = bridge.subscribe((event) => {
         if (closed) return;
         if (isReplaying) {
           if (liveBuffer.length >= LIVE_BUFFER_CAP) {
             bufferOverflow = true;
           } else {
             liveBuffer.push(event);
           }
         } else {
           controller.enqueue(encodeSseEvent("live", event));
         }
       });
       ```

       b. **Handle replay** inside an async IIFE within `start()`:
       ```typescript
       if (hasReplayCursor && eventLog) {
         void (async () => {
           try {
             // Check for stale cursor (per D-04)
             const oldest = await eventLog.oldestSeq();
             if (oldest !== null && sinceSeq < oldest) {
               // Stale cursor — send snapshot signal, skip replay
               controller.enqueue(encodeSseEvent("snapshot", { reason: "cursor_expired" }));
             } else {
               // Capture replay ceiling BEFORE reading file to prevent duplicates
               // Only replay file entries with seq <= ceiling; buffer entries with seq > ceiling
               const replayCeilingSeq = eventLog.currentSeq;

               // Stream missed events from log up to ceiling
               for await (const entry of eventLog.readSince(sinceSeq)) {
                 if (closed) return;
                 if (entry.seq > replayCeilingSeq) break; // Don't read past ceiling
                 controller.enqueue(encodeSseEvent("replay", entry.event));
               }

               // Check for buffer overflow — fall back to snapshot
               if (bufferOverflow) {
                 controller.enqueue(encodeSseEvent("snapshot", { reason: "buffer_overflow" }));
               } else {
                 // Flush buffered live events that arrived during replay (seq > ceiling)
                 for (const buffered of liveBuffer) {
                   if (closed) return;
                   controller.enqueue(encodeSseEvent("replay", buffered));
                 }
               }
             }
           } catch {
             // Log rotation or read error during replay — switch to live gracefully
             if (!closed) {
               controller.enqueue(encodeSseEvent("snapshot", { reason: "replay_error" }));
             }
           } finally {
             isReplaying = false;
             liveBuffer.length = 0;
             // Send live transition sentinel — client dismisses "Catching up..." banner
             if (!closed) {
               controller.enqueue(encodeSseEvent("live", { type: "stream_live" }));
             }
           }
         })();
       }
       ```

       c. **Abort signal cleanup** must also handle readline abort. The `request.signal` abort listener should call `closeWith(controller)` which already calls `unsubscribe?.()`.

    6. **Add ping interval** during live phase to detect zombie connections:
       ```typescript
       const pingInterval = setInterval(() => {
         if (closed) { clearInterval(pingInterval); return; }
         try {
           controller.enqueue(encoder.encode(": ping\n\n"));
         } catch {
           clearInterval(pingInterval);
         }
       }, PING_INTERVAL_MS);
       ```
       Clear `pingInterval` in `closeWith` and `cancel`:
       ```typescript
       clearInterval(pingInterval);
       ```

    7. **Keep backward compat**: When `sinceSeq === -1` (no cursor), behavior is identical to current implementation — live events stream immediately via `encodeSseEvent("live", event)` instead of `encodeSseData(event)`. Existing clients that use `onmessage` will NOT receive named events (they need `addEventListener`). This is intentional — Plan 03 updates the client.
