If the last line is truncated/malformed, keep scanning backward to the last valid JSON line. Handle ENOENT gracefully (fresh log, seq stays 0). This is resilient to crash-truncated files.

       - `append(event: unknown): void` — increments `this.seq`, writes `JSON.stringify({ seq: this.seq, event, ts: new Date().toISOString() }) + "\n"` using `appendFileSync(this.logPath, ...)`.        - `async *readSince(sinceSeq: number): AsyncGenerator<LogEntry>` — if sinceSeq < 0, return immediately (no replay). Opens `createReadStream` + `readline.createInterface`, yields parsed `LogEntry` objects where `entry.seq > sinceSeq`. Handles ENOENT by returning (empty log = no replay). Skips malformed lines silently.

         4. Join the kept lines with `\n` and add trailing `\n`
         5. Write to `this.logPath + ".tmp"`
         6. `rename(tmpPath, this.logPath)` — atomic POSIX rename
         7. Add comment: `// Atomic rename: active readline streams on old inode continue safely (POSIX)`
         No-op if stat throws ENOENT. This ensures whole-line boundaries are preserved.
