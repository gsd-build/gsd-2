## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] React 19 renderToString does not trigger error boundary lifecycle**
- **Found during:** Task 1 (Create ErrorBoundary component and wrap App.tsx)
- **Issue:** The Wave 0 test stub used `renderToString` wrapping a `ThrowingChild`, expecting the ErrorBoundary to catch the error via `getDerivedStateFromError`. React 19 changed this behavior — `renderToString` now propagates the error rather than triggering the boundary lifecycle, causing the test to always throw and fail.
- **Fix:** Rewrote `error-boundary.test.tsx` to directly test the ErrorBoundary class component: call `ErrorBoundary.getDerivedStateFromError(error)` to verify state transition, manually set `eb.state = {hasError: true}` and call `eb.render()` with `renderToString` to verify fallback output.
- **Files modified:** `packages/mission-control/tests/error-boundary.test.tsx`
- **Verification:** 5/5 tests pass (increased from 2 with previous stub)
- **Committed in:** `af1bac6` (Task 1 commit)

**2. [Rule 1 - Bug] animations.test.tsx and sidebar.test.tsx checked for pixel-art SVG output being replaced**
- **Found during:** Task 2 (Replace logo components)
- **Issue:** `animations.test.tsx` checked `LogoAnimationView` for `logo-anim-frame`, `viewBox="0 0 32 32"`, SVG element type; checked `LoadingLogo` for `logo-scan-line`. `sidebar.test.tsx` checked `GsdLogo` for `svg`, `0 0 32 32`, `currentColor`. These tests described the OLD behavior being intentionally removed.
- **Fix:** Updated both test files to assert the new `img`-based output: `gsd-logo-2000-transparent.svg` src, `terminal.svg` src, `animate-pulse` class, `alt` text for accessibility.
- **Files modified:** `packages/mission-control/tests/animations.test.tsx`, `packages/mission-control/tests/sidebar.test.tsx`
