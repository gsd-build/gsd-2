## Decisions Made

- **CORS test isolation:** `addCorsHeaders` in `src/server.ts` is not a standalone export — the file calls `Bun.serve` at the top level. Importing it would start a server during tests. Chose to mirror the function inline so the test is portable and explicitly fails on the wrong value.
- **Wire-guard test strategy:** `wireSessionEvents` is a closure inside `startPipeline`, not exported. Used a standalone simulation of the double-wire pattern to make the test lightweight and fast without spawning real Claude processes.
- **ErrorBoundary rendering:** Project does not have `@testing-library/react` installed. Used `react-dom/server`'s `renderToString` to exercise the class component `getDerivedStateFromError` lifecycle.
- **WsServer hostname access:** The `WsServer` interface does not expose `hostname`. Used `@ts-expect-error` with a clear comment so the test compiles today and the type error becomes a reminder for Plan 01.
