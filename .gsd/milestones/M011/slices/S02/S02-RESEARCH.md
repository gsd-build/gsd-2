# S02: Web CI Workflow & Packaging Verification — Research

**Date:** 2026-03-18
**Status:** Complete

## Summary

This is straightforward infrastructure work. The codebase has a clear CI template (`ci.yml`), an existing packaging validation script (`validate-pack.js`) ready to extend, and S01 has already proven the full `build:web-host` chain works. The slice creates `.github/workflows/web.yml` mirroring `ci.yml`'s structure and extends `validate-pack.js` with web host file checks. No new technology, no ambiguous scope.

## Recommendation

Two tasks: (1) create `web.yml` workflow, (2) extend `validate-pack.js`. Both are small and well-scoped.

**web.yml** should mirror `ci.yml` triggers (push to `main`/`feat/**`, PR to `main`) per D092. Single `ubuntu-latest` runner per D096. Steps: checkout → Node 22 with npm cache → `npm ci` → `npm --prefix web ci` → `npm run build` (builds all workspace packages + tsc) → `npm run build:web-host` (next build + serwist build + stage) → `npm run validate-pack` → `npm run test:unit` → `npm run test:integration`. The web contract tests (`src/tests/web-*-contract.test.ts`) already run as part of `test:unit` via the glob pattern, so no new test commands needed.

**validate-pack.js** needs `dist/web/standalone/server.js` added to the `requiredFiles` array. Also worth adding `dist/web/standalone/public/manifest.json` and `dist/web/standalone/public/sw.js` to confirm S01's PWA artifacts make it into the tarball.

## Implementation Landscape

### Key Files

- `.github/workflows/ci.yml` — Template to copy for web.yml structure. Uses `actions/checkout@v6`, `actions/setup-node@v6` with Node 22 + npm cache. Has `no-gsd-dir`, `build`, and `windows-portability` jobs.
- `.github/workflows/web.yml` — New file. Single job, `ubuntu-latest`, runs the full build + web host build + tests + packaging validation.
- `scripts/validate-pack.js` — Add 3 entries to the `requiredFiles` array: `dist/web/standalone/server.js`, `dist/web/standalone/public/manifest.json`, `dist/web/standalone/public/sw.js`. The script already does `tar tzf` + `includes()` checks — just needs more entries.
- `package.json` — `"files"` field already includes `"dist/web"`, so `npm pack` already captures the staged standalone output. No change needed.
- `scripts/stage-web-standalone.cjs` — Copies `.next/standalone/web` → `dist/web/standalone/`, copies `web/public/` → `dist/web/standalone/public/`. Already handles `sw.js` and `manifest.json` because it copies all of `web/public/`. No change needed.

### Build Order

1. **validate-pack.js first** — adding required files is a 3-line change, immediately verifiable with `npm run validate-pack` locally.
2. **web.yml second** — the workflow depends on the validate-pack extension being in place so CI verifies web packaging.

### Verification Approach

- `npm run validate-pack` exits 0 locally — confirms the new required files are found in the tarball.
- `npm run validate-pack` output shows the 3 new files checked (no `MISSING` lines).
- `.github/workflows/web.yml` exists with correct triggers, steps, and job structure.
- (Operational: pushing to a `feat/**` branch would trigger the workflow on GitHub — not verifiable locally but structurally validated.)

## Constraints

- `web/` is NOT an npm workspace — CI must run `npm --prefix web ci` separately after `npm ci` to install web dependencies.
- `node-pty` in `web/package.json` requires C++ compilation — ubuntu-latest has build-essential pre-installed, no extra apt step needed.
- The `build:web-host` script is `npm --prefix web run build && npm run stage:web-host` — the web build now includes `next build && serwist build` (from S01), which requires `@serwist/cli` + `esbuild` installed in `web/node_modules`.
- `dist/web/standalone/public/sw.js` is a build artifact generated by `serwist build` — it only exists after `build:web-host` runs. The validate-pack step runs after the build steps, so this is fine.
